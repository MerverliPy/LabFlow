# Phase Work

## Use this skill when
- the phase has already been planned
- acceptance criteria are already known

## Workflow
1. Restate the phase objective in one paragraph.
2. Touch only the files needed for this phase.
3. Keep edits local and reversible.
4. Run the smallest useful verification for the touched area.
5. Update STATE.md with what changed.

## Rules
- do not widen scope mid-phase
- avoid unrelated refactors
- prefer one package at a time
- stop and hand off if blockers appear
