# Phase Plan

## Use this skill when
- starting a new phase
- breaking a large change into one safe slice
- you need acceptance criteria before editing files

## Output format
1. Objective
2. Scope
3. Files likely to change
4. Risks
5. Acceptance criteria
6. First exact step

## Rules
- plan only one phase at a time
- keep the plan small enough to finish in one focused session
- prefer stable-core work over experimental work
- call out what will **not** be touched
