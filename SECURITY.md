# Security Policy

## Reporting a vulnerability

Do **not** report suspected vulnerabilities in public GitHub issues.

Use GitHub private vulnerability reporting for this repository if it is enabled.
If private reporting is not enabled, contact the maintainer directly through GitHub.

Include:

- a clear description of the issue
- impact and affected scope
- reproduction steps or a proof of concept
- any suggested mitigation, if available

## Response goals

The project aims to:

- acknowledge receipt of a valid report
- assess impact and reproducibility
- prepare a fix or mitigation
- publish a coordinated update when appropriate

## Scope

Security reports are most relevant for:

- published package behavior
- workflow and automation configuration
- script execution paths
- dependency-related exposure
- state-handling or file-write behavior
