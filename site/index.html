Core commands

Six commands. One coherent workflow.

Each command below shows what it does and gives visitors a real command they can copy immediately.

init

Bootstrap or repair workspace state so the system starts cleanly.

pnpm exec labflow init

task

Create lightweight tasks without jumping into dashboard overhead.

pnpm exec labflow task add "Ship release hardening"

session

Track active work sessions and keep useful continuity around them.

pnpm exec labflow session start

memory

Store local notes and execution context for later retrieval.

pnpm exec labflow memory append "Capture release notes before publish"

status

Expose workspace health, schema state, and proof visibility.

pnpm exec labflow status --json

doctor

Surface environmental drift before it turns into wasted time.

pnpm exec labflow doctor --json

Get started

Start in 60 seconds

This section gives visitors a straight path from landing page to first real command, with copy buttons on every step.

Step 1

Clone and install

Repository setup 
git clone git@github.com:MerverliPy/LabFlow.git
cd LabFlow
pnpm install

Step 2

Build and verify

Baseline checks 
pnpm build
pnpm test
pnpm proof:verify

Step 3

Enter the command surface

First real commands 
pnpm exec labflow --help
pnpm exec labflow init
pnpm exec labflow status --json
  • The first block gets the repo onto the machine.
  • The second block checks that the basics are healthy.
  • The third block gets the visitor into real LabFlow behavior immediately.
